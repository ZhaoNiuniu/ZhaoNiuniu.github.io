Eclipse连接SOL Server 2008

发表于 2019-08-08 |

在连接数据库之前必须保证SQL Server 2008是采用SQL Server身份验证方式而不是windows身份验证方式。如果在安装时选用了后者，则重新设置如下:

修改密码

依次点击：安全性——登录名——双击sa——修改密码为“123456”（或者其他的也行，反正你自己要知道）
修改密码成功后密码行会自动补全，位数不变，不用管它。

再依次点击：sa——状态——启用：

重启：

（1）运行 开始 → 所有程序 → Microsoft SQL Server 2008 → 配置工具 → SQL Server配置管理器，在打开的窗口的左边找到 MSSQLSERVER的协议，在右边右单击 TCP/IP，选择 已启用。如果Named Pipes 未启用也设为启用。如图所示：

（2）双击上图右边的 TCP/IP，在弹出的窗口中选择 IP地址 标签，把 IPAll 中的 TCP端口 设成 1433，并将上方所有的 已启用 选项设置成 是。如图所示：

（3）然后重启数据库。

环境变量CLASSPATH配置：

（1）下载Microsoft JDBC Driver for SQL Server

（从下面地址进去自己搜，现在有6.0的了）
http://www.microsoft.com/zh-cn/download/details.aspx?id=11774 （最新版4.0）
http://www.microsoft.com/zh-cn/download/details.aspx?id=21599 （3.0版）
http://www.microsoft.com/zh-cn/download/details.aspx?id=2505 （2.0版）
下载JDBC的驱动文件，例如下载得到的文件是sqljdbc_4.0.2206.100_chs.exe（可能是6.0），解压文件。设将其解压到C:\Microsoft JDBC Driver 6.0 for SQL Server目录下。（我的是6.0）

（2）在桌面上右击 我的电脑，依次选择 属性 → 高级 → 环境变量，在 系统变量中双击 CLASSPATH变量，追加“;C:\Microsoft JDBC Driver 4.0 for SQL Server\sqljdbc_4.0\chs\sqljdbc4.jar”若不存在应当新建CLASSPATH变量，并且将其值设为“C:\Microsoft JDBC Driver 4.0 for SQL Server\sqljdbc_4.0\chs\sqljdbc4.jar”。如下图所示：（这是网上4.0的示例，是什么版本就自己改）

（3）连续点击 确定 以退出环境变量配置。

使用Eclipse测试连接SQL Server 2008数据库：

（1）打开SQL Server 2008，在其中新建数据库 Test（名字可以自己取，到时候在代码里面名字对应地方改就行，比如我的是shop，我发的代码里面应该也是shop），然后退出SQL Server 2008。

（2）运行Eclipse，新建一个Java Project 名为 Test。

（3）右单击工程名，依次选择 Build Path → Configure Build Path，在打开的窗口的右边选择 Libraries标签，然后单击 Add External JARs，找到 sqljdbc4jar （6.0的是sqljdbc42jar）文件并打开，然后单击 OK 完成构建路径的配置。如图10
